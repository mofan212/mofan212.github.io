1. 范型的引入

在上文 排序算法(一) 中的末尾部分我们提出了一个问题：编写的排序算法只能够对整型数据进行排序，无法做到对浮点型、自定义类型的序列进行排序。为此，我们可以引入范型，增加我们编写的排序算法的适用性。

1.1 父类的修改

在上文 排序算法(一) 中，我们抽取了父类Sort，让我们在编写排序算法时都继承这个父类，以达到代码复用，提高代码的整洁。

要引入范型，我们需要从源头开始修改。因此，我们需要先修改父类。

修改父类很简单，给父类添加范型，然后将成员变量array数组的类型也改成范型，同时将代码中涉及到array数组的变量类型都改为范型。

数组类型修改为范型后，我们会发现我们的比较方法、交换方法出现了问题。交换方法的修改很简单，只需要将tmp的类型修改为范型即可。但是对于比较方法就不适用了，因为引入了范型，我们不知道数据的类型是什么，可能是默认已经有比较性的数据，比如整型、浮点型等，但也有可能是自定义类型，这些类型没有默认有比较性，因此是无法直接使用减法运算进行比较。

但是我们也有办法，我们可以规定进行排序的数据必须具有可比较性，换句话说，范型必须实现Comparable接口，然后在比较方法中，直接调用Comparable接口的compareTo方法进行比较即可。

3. 插入排序

3.1 流程与实现

插入排序（Insertion Sort），非常类似于扑克牌的排序。

插入排序升序排序流程：

1、在执行的过程中，插入排序会将序列分成两个部分

  • 头部是已经排序好的，尾部是待排序的

2、从头开始扫描每一个元素

  • 每当扫描到一个元素，就将它插入到头部合适的位置，使得头部数据依然保持有序

稳定性

我们发现，插入排序打印出来的稳定性是稳定的。其实我们可以分析一下：

假设现有一个序列{10a, 10b, 5, 17, 9}，前两个数是都是指数字10，加个字母便于区分。

在插入排序中，我们的比较代码是：cmp(cur, cur - 1) < 0，表示后者小于前者时才进行交换。对于给出的序列，10b 是等于 10a 的，并不是小于 10a 的，因此不会进行交换，继续进行插入排序，最后 10a 与 10b 的相对位置不会发生改变，因此插入排序是一个稳定的排序。

3.2 逆序对

逆序对（Inversion），那么什么是逆序对呢？

设 A 为一个有 n 个数字的有序集 (n>1)，其中所有数字各不相同。

如果存在正整数 i, j 使得 1 ≤ i < j ≤ n 而且 A[i] > A[j]，则 <A[i], A[j]> 这个有序对称为 A 的一个逆序对，也称作逆序数。

现有一个数组<2,3,8,6,1>，那么这个数组的逆序对为：<2,1>，<3,1>，<8,1>，<8,6>，<6,1>，共 5 个逆序对。

插入排序的时间复杂度与逆序对的数量成正比关系。

简单来说就是：逆序对的数量越多，插入排序的时间复杂度越高。

那为什么呢？

在上图中，最开始给出了一个待排序序列，我们需要使用插入排序对这个序列进行升序排序，最终的排序结果是在最后一行。

待排序列是一种逆序对最多的情况，我们进行插入排序时，每一个插入元素要和左边已经排序好的所有元素进行比较，最后会把这个插入元素放在左边已排序好序列的首位。

那么上述序列使用插入排序进行升序排序的时间复杂度为多少呢？

在第一行中，交换一次；第二行中，交换两次；第三行中，交换三次，到第 n 行时，交换 n 次。时间复杂度就是交换次数之和，使用大O表示法的时间复杂度为 O(n2)。（时间复杂度其实也相当于蓝绿色三角形的面积）

插入排序时间复杂度

最坏、平均时间复杂度： O(n2)

最好时间复杂度： O(n)

对一个序列使用插入排序进行升序排序时，如果这个序列本来就是升序的，那么这时候插入排序的时间复杂度为：O(n)。

空间复杂度： O(1)

插入排序属于稳定排序。

当逆序对的数量极少时，插入排序的效率特别高，甚至速度比 O(nlogn) 级别的快速排序还要快。

数据量不是特别大时，插入排序的效率也是非常好的。

3.3 插入排序优化一

我们最开始使用插入排序时，使用的是【交换】，我们可以将【交换】转为【挪动】。

具体步骤是：

1、先将待插入的元素进行备份

2、头部有序数据中比待插入元素大的，都朝尾部方向挪动 1 个位置

3、将待插入元素放到最终的合适位置

while循环执行的次数多，就表明交换的次数（未优化前）或挪动的次数（优化后）越多，这两种情况越多其实就表明数组中的逆序对越多，因此可以得出逆序对越多，插入排序效率越低。这又一次验证了： 逆序对的数量越多，插入排序的时间复杂度越高。

4. 二分搜索

4.1 需求的提出

我们应该怎么确定一个元素在数组中的位置呢？（假设数组里面全都是整数）

如果是一个无序数组，从第 0 个位置开始遍历搜索，平均时间复杂度为： O(n)

如果是有序数组，可以使用二分搜索（Binary Search），最坏时间复杂度为： O(logn)

4.2 思路讲解

对一个升序数组，使用二分搜索：

假设在 [begin, end)范围内搜索某个元素 v ，mid == (begin + end) / 2

PS： end 表示数组长度，而不是数组的最大下标。

  • 如果 v < m ，去 [begin, mid) 范围内二分搜索
  • 如果 v > m ，去 [mid + 1, end) 范围内二分搜索
  • 如果 v == m ，直接返回 mid

从上面的示例中我们在设计 end 时，我们是将数组的长度赋值给 end 的；而 begin 就是数组的第一个索引 0 。

因此， end 是一直大于 begin 的，当 end 突然等于 begin 的时候，就表明查找的元素在数组中不存在。😏

数组的长度会比数组的最大索引大 1 ，那为什么要设置成数组长度呢？不直接设置成最大索引呢？

如果我们想要求得从 begin 到 end 之间有多少个元素时，我们只需要使用end - begin即可。如果我们将 end 设计成数组最后一个索引，那么再求 begin 和 end 之间的元素个数时，就需要end - begin + 1，这样做岂不是很麻烦？😰

从这里也可以得出一个启发：我们在设计区间范围的时候，尽量是左闭右开的，这样会让我们的编码变得简单易懂。😝

4.4 获取插入位置

插入排序优化分析

前文介绍了插入排序和二叉搜索，那这两个有什么关系吗？

插入排序对一组乱序数据进行排序，二叉搜索是对一组排序好的数据进行查找，怎么看都没关系啊。😟

非也非也。还记得插入排序会把序列分成两部分吗？一部分是已经排序好的，另一部分是待排序的，然后我们需要从待排序的那一部分中挑选一个元素将其插入到已经排序好的序列中，这时我们需要确定插入的位置，而在这，就可以使用二分搜索进行查找。

按照最开始的做法是一个一个进行比较，然后交换，交换代码过多效率不行，我们对其进行优化；然后我们依然是一个一个进行比较，但是这个时候不进行交换了，改成挪动，这样会减少代码量以提高效率。但是我们发现，进行优化后的效率虽然提升了，但是不明显，这个时候我们可以使用二分搜索，减少比较次数。

使用二分搜索的话，我们前面编写的二分搜索代码是不行的，主要原因有二：

  • 如果要将一个元素插入到已经排序好的序列中，插入元素是不存在于排序好的序列中的，使用以前的代码就会返回 -1 ，这显然是憨憨嘛
  • 同时，如果我插入一个在已排序好的序列中存在的数据，使用以前的代码会返回这个数据的索引，然后我用新数据覆盖旧数据？

因此使用最开始编写的代码肯定是不行的，那么我们应该怎么做呢？

在元素 v 的插入过程中，我们可以先二分搜索出合适的插入位置，然后再将元素 v 插入。

要求二分搜索返回的插入位置： 第一个大于 v 的元素位置。

如果 v 是 5，返回 2

如果 v 是 1，返回 0

如果 v 是 15，返回 7

如果 v 是 8 ，返回 5

获取插入位置思路

假设在 [begin, end)范围内搜索某个元素 v ，mid == (begin + end) / 2

PS： end 表示数组长度，而不是数组的最大下标。

  • 如果 v < m ，去 [begin, mid) 范围内二分搜索
  • 如果 v >= m ，去 [mid + 1, end) 范围内二分搜索

为了保证插入排序的稳定性，当插入元素与 m 相等时，我们要找到第一个大于插入元素的位置，然后我们应该将插入元素插到那个位置处。

尽管我们使用二分搜索来查找元素插入的位置后效率得到了显著提升，但我们需要注意的是，使用了二分查找后，只是减少了比较次数，插入排序的平均时间复杂度依然是 O(n2)。

使用二分搜索查找插入元素的位置的时间复杂度为 O(logn) ，但是找到插入位置后，我们还要进行时间复杂度为 O(n) 级别的挪动操作。

这还只是一个元素的插入挪动，我们拥有 n 个元素，所以最终的平均时间复杂度还是O(n2)。

时间复杂度是没有改变的，我们只是在原有的时间复杂度的基础上做了优化，我们在测试结果中也可以看到，优化后的插入排序依旧比不上时间复杂度为O(nlogn)的堆排序。（堆排序流弊！！💃）
