图片处理

一篇好的文章讲究什么？图文并茂！这样可以使读者更好地理解文章内容。

在 Typora 中，或者说 Markdown 语法中，图片都是引用的，并不是直接嵌入文章的。其实很好理解，就和 HTML 中的<img>一样，设置一个路径，然后图片就可以显示了，如果路径错误或者图片不存在，那图片就无法显示。

而对于 Typora 来说，在文章里引用图片是十分简单的，你可以直接将图片拖拽进文章，Typora 会自己定位图片的绝对路径并引用图片。当然，如果你截了图，你可以直接在 Typora 中复制，截取的图片可以直接在 Typora 中引用，而图片会保存在用户路径下。

为了方便图片的整理，我一般将文章分成不同的类，每类文章中有保存的图片文件夹。比如，这篇文章我将其分入Tools类别中，那么这个类别下就会有专门存放这类文章图片的文件夹：

这样的话可以方便我进行寻找图片、替换图片或修改图片。

针对这一点，我对Typora保存图片进行了以下设置，这Typora顶部菜单栏找到 格式 一项：

  • 格式 — 图像 — 设置图片根目录…
  • 格式 — 图像 — 当插入本地图片时 — 复制图片到文件夹…

我为什么要进行这些设置？ 且听我娓娓道来~

设置图片根目录

Q : 为什么要设置图片根目录？

A : 设置了图片根目录会让你图片的引用路径从绝对路径变成相对路径。

Q : 那这有啥用？

A : 你看，我是将每类文章都设置了不同的图片存放文件夹，并且文章和图片存放文件夹处于同一目录下，如果我想要将我的文章转移目录，那么直接剪切就可以了，不用修改图片引用路径，图片照样可以显示，这就是设置了根目录的魅力！ 😎

A : 而且对于我个人来说，我喜欢将重要的东西进行备份，比如文章！ 要备份的话，可以存入 U 盘，然后能在不同的 PC 上查看（当然你也可以传到自己的博客），还可以备份到云上。如果设置了绝对路径，直接在U盘中打开文章也可以直接看到图片，备份到云上后下载下来也可以直接看到图片。这就是！ 设置了！根目录的！魅力！ 🎉

复制图片到文件夹

Q : 为什么要设置 “复制图片到文件夹”？

A : 如果你是截图然后复制到Typora中，Typora会将这些图片保存到用户目录（C盘下的一个目录）下，但是C盘寸土寸金，我可不舍得。

A : 所有图片都挤在这样一个目录下，我觉得不方便查找或修改，如果你设置了“复制图片到文件夹”，这时候你再次截图然后复制到Typora中，这些图片就会保存在你设置的文件夹中，这样就可以实现图片分类整理。

A : 如果将本次设置和上一步设置搭配使用，就可以将截取的图片存放到指定的文件中，并且图片的引用还是相对路径，文章、图片的备份就会显得异常简单，直接将最外层大文件夹拖拽进 U 盘或上传至云上即可。 那是相当的奈斯啊！👍

Q : 我使用picGo不好吗？直接将图片传到图床，干嘛还将图片留到本地？

A : 这就是个人习惯的问题了，首先图床这个东西我不是绝对信任，而且我有备份的习惯，感觉在本地保留一份（或多备份几份）是必要的。同时，我本人有“命名洁癖”，我不能忍受图片的名字是一串没有意义的字符，我习惯将它们命名成有意义的，这也方便后续修改。如果使用了picGo后，上传到图床上的图片名字都是没有意义的字符，emmmm，我觉得⑧行，因此就算麻烦点，我还是选择手动修改名字后再手动上传到图床。

总的来说，这是我个人使用的一些习惯，你可以跟着我的习惯来，如果你觉得我这样有点麻烦，你也可以使用其他的方式。

Typora是一款简介、高自定义化的Markdown编译器，你一定能找到适合自己的一项设置。

本语法适用于在Typora编译器， 不同的编译器对Markdown语法的编译可能不同，但是大同小异。

图片

Markdown 图片语法格式如下：

  • 开头一个感叹号 !
  • 接着一个方括号，里面放上图片的替代文字
  • 接着一个普通括号，里面放上图片的网址，最后还可以用引号包住并加上选择性的 ‘title’ 属性的文字。

当然，你也可以像网址那样对图片网址使用变量。

Markdown 还没有办法指定图片的高度与宽度，如果你需要的话，你可以使用普通的 <img> 标签。比如：<img src="xxxxxxxxx" width="50%">

表格

Markdown 制作表格使用 | 来分隔不同的单元格，使用 - 来分隔表头和其他行。

语法格式如下：

表头 表头
单元格 单元格
单元格 单元格

我们可以使用表格上方的对齐方式调整表格的对齐方式，也可以在书写时调整表格的对齐方式。

  • -: 设置内容和标题栏居右对齐。
  • :- 设置内容和标题栏居左对齐。
  • :-: 设置内容和标题栏居中对齐。

首行缩进

写文章时，我们常常希望能够首行缩进，这时可以在段首加入&ensp来表示一个空格，或者使用&emsp表示两个空格。

为什么要使用这种方式，因为在不同的Markdown编译器中，直接输入空格或者制表符可能无法显示。虽然在Typora上可以正常显示，但是将.md文件传到个人博客之类的支持Markdown语法的网站可能会失效。
