图片处理

在 Typora 中，或者说 Markdown 语法中，图片都是引用的，并不是直接嵌入文章的。其实很好理解，就和 HTML 中的 <img> 一样，设置一个路径，然后图片就可以显示了，如果路径错误或者图片不存在，那图片就无法显示。

而对于 Typora 来说，在文章里引用图片是十分简单的，可以直接将图片拖拽进文章，Typora 会自己定位图片的绝对路径并引用图片。各种截图可以直接在 Typora 中复制，截取的图片可以直接在 Typora 中引用，它们会保存在用户路径下。

为了方便图片的整理可以将文章分成不同的类，每类文章中有单独存放的图片文件夹，以便快速寻找、图片或修改图片。比如：

针对这一点，需要对 Typora 进行以下设置，这 Typora 顶部菜单栏找到 格式 一项：

  • 格式 — 图像 — 设置图片根目录…
  • 格式 — 图像 — 当插入本地图片时 — 复制图片到文件夹…

设置图片根目录

Q：为什么要设置图片根目录？

A：设置了图片根目录会让图片的引用路径从绝对路径变成相对路径。

Q：那这有啥用？

A：每类文章都设置了不同的图片存放文件夹，并且文章和图片存放文件夹处于同一目录下，如果需要转移文章目录，直接拖动文件到目标位置即可，无需修改图片引用路径，这就是设置了根目录的魅力！ 😎

A：除此之外可以更方便地备份，比如存入 U 盘，然后在不同的 PC 上查看，还可以备份到云上。绝对路径相比于相对路径，直接在 U 盘中打开文章也可以直接看到图片，备份到云上后下载下来也可以直接看到图片。这就是！ 设置了！根目录的！魅力！ 🎉

复制图片到文件夹

Q：为什么要设置“复制图片到文件夹”？

A：截图后自接复制到 Typora 中时，Typora 会将这些图片保存到用户目录（C 盘下的一个目录）下，不方便管理。在设置了“复制图片到文件夹”后，这些图片就会保存在设置的文件夹中，实现图片分类整理。

A：如果将本次设置和上一步设置搭配使用，就可以将截取的图片存放到指定的文件中，并且图片的引用还是相对路径，文章、图片的备份就会更加简单，直接将最外层文件夹拖拽进 U 盘或上传至云即可。那是相当的奈斯啊！👍

Q：使用 picGo 不好吗？直接将图片传到图床，干嘛还将图片留到本地？

A：这就是个人习惯的问题了，首先图床这个东西并不是绝对信任的，养成备份的习惯，更能有备无患，在本地保留一份（或多备份几份）是必要的。

Markdown 语法

本语法适用于在 Typora 编译器， 不同的编译器对 Markdown 语法的编译可能不同，但大同小异。

标题

  1. 可以使用 # 表示 1-6 级标题，一级标题对应一个 # 号，二级标题对应 ## 号，依次类推。

段落

  1. 段落的换行是使用两个以上空格加上回车
  2. 或者输入完内容后直接回车即可

分割线

可以在一行中用三个以上的星号、减号、底线来建立一个分隔线，行内不能有其他东西，也可以在星号或是减号中间插入空格。

Markdown 支持有序列表和无序列表。

无序列表使用星号 *、加号 + 或是减号 - 作为列表标记：

列表嵌套只需在子列表中的选项添加四个空格即可：

    • 第一项嵌套的第二个元素

图片

Markdown 图片语法格式如下：

  • 开头一个感叹号 !
  • 接着一个方括号，里面放上图片的替代文字
  • 接着一个普通括号，里面放上图片的网址，最后还可以用引号包住并加上选择性的 title 属性的文字。

Markdown 还没有办法指定图片的高度与宽度，如果需要，你可以使用普通的 <img> 标签。比如：<img src="xxxxxxxxx" width="50%">

表格

Markdown 制作表格使用 | 来分隔不同的单元格，使用 - 来分隔表头和其他行。

语法格式如下：

表头 表头
单元格 单元格
单元格 单元格

可以使用表格上方的对齐方式调整表格的对齐方式，也可以在书写时调整表格的对齐方式。

  • -: 表示设置内容和标题栏居右对齐。
  • :- 表示设置内容和标题栏居左对齐。
  • :-: 表示设置内容和标题栏居中对齐。

首行缩进

可以在段首加入 &ensp 来表示一个空格，或者使用 &emsp 表示两个空格。

为什么要使用这种方式，因为在不同的 Markdown 编译器中，直接输入空格或者制表符可能无法显示。虽然在 Typora 上可以正常显示，但将文件传到个人博客之类的支持 Markdown 语法的网站可能会失效。

单等号对齐：

KPI=(N+S)W=NW+SW\begin{aligned} KPI &= (N + S)W \\ &= NW + SW \end{aligned} KPI​=(N+S)W=NW+SW​

多公式等号对齐：

KPI=(N+S)WPI=N+SI=W\begin{aligned} KPI &= (N + S)W \\ PI &= N + S \\ I &= W \end{aligned} KPIPII​=(N+S)W=N+S=W​

KPI=(N+S)WPI=N+SI=W\begin{aligned} &KPI = (N + S)W \\ &PI = N + S \\ &I = W \end{aligned} ​KPI=(N+S)WPI=N+SI=W​
